# 我们的项目到底在做什么？——写给人文学者的完整解释

*依据：7 月 17 日三人会议记录 + 德乾 7 月 13 日的 formulation note + 给 Jordan 的群回复。*
*用途：帮助非计算机背景的合作者理解项目现状、德乾提出的改造方案、以及接下来各自要做什么。*

---

## 一、一分钟版本

我们手里有两堆图：**档案原照**（晋察冀语料，四万多张）和**杂志印刷品**。任务是找出"哪张杂志图印的是哪张原照"。

- **现在的做法**：一条四道关卡的"流水线"，每道关卡用人手定的门槛值把候选一层层筛掉，最后剩下的给专家（您）过目。它**好用，但说不清**——审稿人会问"门槛为什么是 50？为什么是 0.65？"，我们只能回答"试出来的"。
- **德乾的方案**：整条流水线**原封不动保留**，但降级为两个配角——"介绍人"（提供候选配对）和"体检中心"（给每对候选出一份证据报告）。真正做判断的换成一个**用我们已有的专家标注训练出来的小模型**；原来的手工门槛规则保留下来，作为跟新模型打擂台的"最强手工基线"。
- **故事也换了**：论文不再讲"我们造了一条筛选流水线"（这种应用型故事很难被 AI 会议接受），而是讲"我们把历史图像的配对定义成一个**搜索/估计问题**，用小规模专家标注 + 学习模型来解它，并且给出错误率有保证的人机分工方案"。
- **对您最实际的影响**：您的人工标注从"判对错"升级为**三分类**（同一底片 / 同一场景但不同底片 / 无关），并且需要对已判"不匹配"的部分做一次**复核**。后面几批的标注不用赶，德乾原话："挑 2 批和挑 6 批只是论文里数字的区别，投稿后再补也来得及。"

## 二、现状：四道关卡的流水线

把它想象成一个漏斗。以第一批数据（Shard 1）为例：

```
约 7,000 张原照
   │  关卡一：DINO —— 每张原照找出"长得最像"的 10 张杂志图
   ▼
约 70,000 对候选
   │  关卡二：ASpanFormer —— 数两张图之间能对上的"关键点"，少于 50 个的淘汰
   ▼
约 650 对幸存者   ←—— 专家（您）在这里人工过目，约一半是真配对
   │  关卡三：VGGT —— 算几何证据（内点率、相机位姿），低于手工门槛（0.65 / 2.13）的淘汰
   ▼
最终输出的"高置信配对"
```

三道技术关卡各自在看什么，可以这样类比：

| 关卡 | 工具 | 它在回答的问题 | 类比 |
|---|---|---|---|
| 一 | DINO | 这两张图**整体气质**像不像？ | 媒人凭第一印象拉名单 |
| 二 | ASpanFormer | 两张图上能找到多少**能一一对上的细节点**？ | 对指纹：对上的点越多越可疑是同源 |
| 三 | VGGT | 这些对上的点在**几何上**是否自洽？两张图是不是同一台相机、同一个机位拍的？ | 法医鉴定：不光看像不像，还原拍摄现场 |

这条流水线本身运转良好——按目前的统计，真配对一旦活着到达关卡三，被正确放行的比例约 97%。**问题不在效果，在"说法"。**

## 三、麻烦在哪里：三个说不清

1. **门槛值说不清。** 50 个关键点、0.65 的内点率、2.13 的位姿分，都是 Jordan 试出来的。审稿人问"依据是什么"，我们没有原则性的回答。其中**关卡二的 50 最难办**：它把 70,000 对砍到 650 对，而被砍掉的那 69,000 多对，谁也不可能全部人工看一遍来证明"砍得对"。
2. **没有学习成分。** 对 AI 会议来说，一条纯手工调参的管线，无论多好用，都容易被归为"工程应用"而不是"研究贡献"。德乾判断这样投稿"风险太高"。
3. **故事不占理。** "过滤管线"听起来是产品说明书；而"把历史图像考据定义成一个可以被算法求解的搜索问题"才是一个研究问题——而且将来可以自然延伸到版画、木刻、甚至跨媒介（图配文）的场景，这正是您在会上感兴趣的方向。

## 四、德乾的重构方案（note 的核心思想）

### 4.1 换一个画面：从"漏斗"到"一张网"

忘掉漏斗，想象一张巨大的关系网：**每张图是一个点**——不管它是档案原照还是杂志印刷品；**两张图若同源，就连一条线**。我们要还原的其实是两层嵌套的"家谱"：

- **同一次曝光**（同一张底片）：这是我们最终要的"真配对"；
- **同一个场景**（同一天同一地点的拍摄，但快门不是同一次）：会上 Jordan 举过的例子——炮手在大炮前，另一张里他往前走了两步。两张图**极像但不是同一张底片**，这正是以前最容易判错的一类。

这个画面还白送一个好处：**传递性**。如果 A 和 B 是同一底片、B 和 C 是同一底片，那 A 和 C 必然是同一底片——不用任何人再看一眼。德乾在 note 里算过：光靠目前 577 个已确认正例，通过这种"连坐推理"就能免费多确定 195 条关系。这就是德乾在会上说的"朋友圈"比喻：确认了几对朋友关系，就能推断出更多人际连线。

### 4.2 原有管线去哪了？——降级，但一行代码都不改

| 原来的身份 | 新的身份 |
|---|---|
| 关卡一（DINO 找前十） | **介绍人**：负责提出"哪些配对值得考察"，冻结不动 |
| 关卡二（关键点计数） | 介绍人的一部分：候选名单的最后一道准入线，冻结不动 |
| 关卡三（VGGT 算几何） | **体检中心**：给每对候选出一份"证据报告"，冻结不动 |
| 手工门槛规则（0.65 / 2.13） | **打擂台的对手**：作为"最强手工基线"保留，新模型必须赢过它才算数 |

"证据报告"（note 里写作 g_ij）就是每对候选的一张体检单，上面十来项指标全部**已经算好、存在现成文件里**：对上的关键点数量、几何误差的均值中位数、重叠比例、内点率、相机位姿差异、整体相似度等等。**不需要跑任何新计算。**

### 4.3 新的判断者：一个小模型

Jordan 最初的设想是训练一个 MLP（可以理解为"一个会自动学权重的打分公式"）。德乾 note 里的版本更进一步但精神一致：一个跑在上述关系网上的小型图网络（代号 NEG-Net），它做的事情说白了就是——

> 读每对候选的体检单，同时**参考网络里邻居们的判断**（既然 A=B、B=C 基本坐实，那 A=C 就不该给低分），输出两个概率：这对是不是同一场景？是不是同一底片？

它约两百万个参数，在深度学习里算**非常小**——这是刻意的，因为我们只有几百对专家标注，模型大了就会"死记硬背"。这也正是德乾反复说"半监督"的意思：模型在整张网（含大量无标注配对）上运转，但只在有专家标注的地方对答案。如果连这个小模型都训不动，德乾的保底方案是退回到严谨的线性回归——本质上就是"给体检单上每项指标学一个权重"，故事依然成立。

### 4.4 为什么标签要分三类（与您直接相关）

以前的标注是二分的：匹配 / 不匹配。新方案要求三分：

1. **同一底片**（真配对）；
2. **同场景不同底片**（大炮前走两步）；
3. **无关**。

原因：第 2 类以前被混在"不匹配"里当反例喂给规则，可它偏偏长得最像正例——把它单独立为一类，模型才能学会这个最微妙的区分，而这恰恰是专家肉眼的看家本领。把已有标注中的"不匹配"重新分成第 2、3 类（顺带抓漏网的真配对），就是所谓的**负例复核（negative audit）**。代码上已经设计好：复核结果随时可以像"换一版数据"一样接进去，不需要推倒重来，所以**不急，但重要**。

### 4.5 最难的问题（"50 怎么辩护"）的解法：不辩护，改测量

这是整个 note 里最漂亮的一步棋。思路转换是：

> **旧思路**：证明"50 是对的门槛" → 做不到，因为没人能看完 69,000 多对被淘汰的候选。
> **新思路**：把"介绍人"（关卡一 + 关卡二）宣布为**给定的固定前提**，不声称它最优，转而**测量它漏掉了多少**。

具体地，最终查全率被拆成两段相乘：

**端到端查全率 = 漏斗覆盖率 × 决策查全率**

- **决策查全率**（真配对进了候选名单后，被正确放行的比例）：这是模型负责的部分，有专家标注，能严格测量——目前约 97%。
- **漏斗覆盖率**（真配对能进候选名单的比例）：没法直接测，但可以**抽样审计**——(a) 从被淘汰的配对里分层抽一小部分请专家复核（不需要看全部 7 万对）；(b) 用**建模之前**人工考据找到的那批晋察冀配对当"考题"，看介绍人能不能把它们招回来。

这样论文里就不存在"请相信 50 是对的"这句话了。淘汰造成的偏差（术语叫"发现偏差"）从需要辩解的软肋，变成一个**被定义、被测量、被如实报告的量**。

### 4.6 最后一步：模型不拍板，人拍板

模型输出的是概率，不直接说"是/否"。最外层有一个校准过的决策规则，把每对候选分到三个筐：

- **接受**（几乎确定是真配对）；
- **送人工复核**（拿不准的，请专家看）；
- **拒绝**（几乎确定无关）。

并且带数学保证：**真配对被误扔进"拒绝"筐的比例不超过事先设定的上限**，同时让需要专家过目的量尽可能少。对历史学界这是最诚实的产品形态：机器不取代专家判断，只是把专家的时间用在刀刃上，并保证漏检率可控。

## 五、数据纪律：六批数据，各司其职

数据一共分了六批（shard）。角色分配：

| 数据 | 角色 | 纪律 |
|---|---|---|
| Shard 1 | **教材**：训练模型 | 内部再切分时按"曝光家族"整体划分，防止同源图片泄题 |
| Shard 2 | **模拟考**：验证与调整 | 标注已完成，冻结使用 |
| Shard 3 | **正式高考**：一次性盲测 | 方案定稿前**绝不碰**，只考一次 |
| Shard 4–6 | 后备 | 标注可慢慢补，不影响主线 |
| JOCCH 数据集（之前与 Brandon 合作的几千张） | **换个考场再考一次**：验证方法是否只对晋察冀有效 | 全程不碰，一切冻结后做唯一一次零样本测试 |

"不能偷看"不是仪式感：一旦在调参时瞟过考卷，这门考试的成绩就永远失去说服力（术语叫"数据污染"）。Jordan 在会上主动提出对 JOCCH 保持隔离，德乾把决定权交给你们二位，群回复里我们已确认支持。

## 六、Jordan 问的三个技术问题，翻译成人话

1. **"模型结构到底用什么？"** —— 不是独立的一个 MLP，而是 4.3 节那个跑在关系网上的小模型；Jordan 设想的 MLP 没有被否定，而是变成了这个小模型读体检单的那个部件。具体每层多宽，由 Jordan 在实验计划里提方案、德乾把关。
2. **"关键点的原始配对数据（一长串坐标）要不要直接喂给模型？"** —— 不喂原始数据，喂**从它提炼出的固定几项统计指标**——而这些指标就是体检单上现成的那些，一项新计算都不用跑。原因：原始坐标串长短不一，处理它需要大得多的模型，我们几百条标注养不起。
3. **"内点率要不要预先算好传进去？"** —— 要。它本来就是体检单的第一项。让模型自己从原始数据里重新推导内点率，是浪费我们本就稀缺的标注。总结：**内点率进、统计指标进、原始坐标串不进。**

## 八、小词典（按出场顺序）

| 术语 | 人话 |
|---|---|
| DINO | 一个看图提"气质特征"的现成 AI 模型，用来做第一轮海选 |
| 特征向量 / embedding | 把一张图压缩成一串数字，数字越接近的图长得越像 |
| 关键点（keypoint） | 两张图上能一一对应的细节位置，类似指纹比对里的比对点 |
| 内点率（inlier ratio） | 对上的关键点里，几何上"说得通"的占比；越高越可能同源 |
| 位姿（pose） | 算法反推出的相机位置和朝向；同一底片的两个印本应反推出几乎同一个机位 |
| MLP | 最简单的一种神经网络，可理解为"自动学权重的打分公式" |
| 图网络 / NEG-Net | 跑在"点和线"关系网上的神经网络，打分时会参考邻居关系；NEG-Net 是德乾给我们这个小模型起的代号 |
| 半监督学习 | 少量有标注数据 + 大量无标注数据一起用：模型在全网上运转，只在有标注处对答案 |
| 传递闭包 | "A=B 且 B=C 则 A=C"式的连坐推理，能免费扩充已知关系 |
| 负例复核（negative audit） | 把已判"不匹配"的样本重新细分（同场景？无关？），并抓漏网正例 |
| 发现偏差（discovery bias） | 我们的标注只覆盖"介绍人推荐过的候选"，没被推荐的真配对天然不可见——新方案把它定义出来并测量，而不是回避 |
| 零样本（zero-shot） | 模型从没在某数据集上训练或调整过，直接上去考试 |
| conformal / 风险控制 | 一类统计方法，能对"接受/复核/拒绝"的错误率给出数学上限 |
| 消融实验（ablation） | 把系统的某个零件拆掉再测，看它到底有没有贡献（Jordan 用它发现关卡一的自训练小头可以去掉） |

---

*一句话收尾：管线不动、代码不改、标注升级、说法重构——把"我们手工调了一条好用的流水线"，变成"我们定义并求解了一个历史图像搜索问题，且错误率有保证"。*

# 进展更新（7 月 18 日）

前面的正文写于德乾书面反馈之前。这一节记录之后发生的三件事：**德乾对实验方案的五点拍板**、**Jordan 发现的一个数据划分隐患及其处理**、以及**代码初稿已经写出来了**。

## 一、德乾拍板的五件事（逐条说明白）

**1. 每张图用哪种"数字指纹"喂给模型：用加工前的，不用加工后的。**
每张图都会被转换成一长串数字（特征）才能进入模型。我们有两个版本可选：一个是三个现成识别模型的输出直接拼起来的"原始版"（3024 个数字），一个是再经过我们自己训练的一个小部件加工后的"精简版"（1000 个数字）。德乾定了用**原始版**。理由很实际：那个加工用的小部件正是 Jordan 的消融实验发现"可有可无、可能要删掉"的部件——如果用精简版，将来删掉它，所有数据就得重做一遍；用原始版则不受影响。代价只是 Jordan 要写个小脚本把全部图片过一遍，一次搞定。

**2. 模型结构定型。**
Jordan 原来设想的那个简单打分模型（MLP）没有被推翻，而是成为整个模型读取"每对候选证据"的那个部件。具体尺寸德乾都给了默认值，Jordan 写进实验计划后他把关。其中有一条要求特别值得您了解：模型必须保留一个开关，能把"参考邻居关系"的功能**整个关掉**——关掉之后就退化成 Jordan 最初设想的简单版本。为什么？因为论文里必须回答"性能提升到底是新结构带来的，还是仅仅因为模型变大了"。有了这个开关，我们就能报告一张递进对比表：手工规则 → 简单模型 → 加上邻居参考 → 加上一致性约束，每一步涨了多少，一目了然。审稿人最认这种"逐项归因"的证据。

**3. 训练的评分标准（损失函数）：先简后繁。**
先用最标准的方案（外加一条"一致性惩罚"：如果 A=B、B=C 都很确定，模型却说 A≠C，就扣分——把我们之前说的连坐推理变成训练时的软性约束），跑通并出基线成绩后，再上德乾 note 里的完整方案。这和我们群里之前建议的顺序一致，德乾书面确认了。

**4. 数据纪律维持原议。**
Shard 2 冻结验证、Shard 3 一次性盲测、JOCCH 全部冻结后再做零样本测试——与前文第五节的表格完全一致，不再有争议。

**5. 一个新概念："先验校正"。**
这是五条里唯一的新东西，值得花一段说清楚。我们的训练数据里，真配对约占一半（Shard 1 是 50.7%）；但 Shard 2 里只占 39.5%；将来真正用在整个档案上时，真配对的比例会低得多得多——毕竟绝大多数"档案照片×杂志图"组合是无关的。模型是在"对错各半"的环境里学出打分习惯的，直接搬到"对的很稀少"的环境，它会**系统性地过于乐观**——就像按训练环境的比例去押注，到了新环境必然押多了"是"。解决办法是一个标准的数学修正：按两边比例的差距，给模型的所有打分做一次**固定幅度的下调**，然后才轮到划线、校准等后续步骤。不做这一步，后面"接受/复核/拒绝"的错误率保证就建立在歪的地基上。这个修正不需要发明，教科书里有现成公式，代码里已经写好了。
