# 第一部分：这个项目在做什么

## 1.1 研究问题

我们有两批图像：

- **档案原照**：华北交通档案（華北交通アーカイブ，日本 CODH/ROIS 数字化），42,773 张。
- **杂志印刷品**：《北支》（*Hokushi*，1939–1942）杂志页面上的照片，2,856 张。

任务：找出杂志上印的每一张照片，翻印自档案里的哪一张原照。

这在史学上叫**溯源（provenance）**。做成了，历史学家就能知道某张宣传照片的原始拍摄记录、以及它在印刷时被如何裁切、修版、重新语境化。

## 1.2 为什么这件事难

三个技术困难，缺一不可地构成了这个问题的难度：

1. **印刷品和原照长得不一样**：印刷经过裁切、网点加网（halftone）、修版，图像的像素级信息已经严重改变。
2. **档案里全是近似重复**：摄影师在同一个场景连按几次快门，得到几张几乎一样但**不是同一张底片**的照片。判断"是同一张底片的翻印"还是"是同场景另一张底片"，是这个任务的核心难点。
3. **规模**：42,773 × 2,856 意味着超过一亿个可能的配对。人工穷举不可能。

第 2 点特别重要：如果把"同场景另一次快门"误判成"同一张底片"，历史学家会据此写出错误的史实关联。这不只是漏掉一个匹配，而是**制造伪证**。这也是论文标题的由来：*Same Negative or Same Scene?*

## 1.3 项目的两个阶段

**阶段一（本项目开始前已完成）**：Jordan 建了一条四步流水线，用现成的 AI 模型逐层筛选候选配对，最后用手工设定的阈值判定。它能用，效果不错，但在方法学上说不清楚：阈值为什么是这些数字？漏掉了什么？有多确信？

**阶段二（本项目，7 月 17 日会议启动）**：孔德乾提出把整件事**重新形式化**——不再把它描述为"一条过滤流水线"，而描述为一个**估计问题**：我们要恢复的是图像之间的一种隐藏结构；流水线降级为"提供候选和证据"的固定前置装置；判定环节换成一个用专家标注训练的模型；并且给出错误率的数学保证。

本文档主要讲阶段二。

# 第二部分：现有系统（阶段一）如何运作

理解后面所有决策的前提，是先理解这条流水线。它有四步：

| 步骤 | 用的工具 | 在做什么 | 通俗解释 |
|---|---|---|---|
| 1. 检索 | DINOv3 | 每张档案照片找出最相似的 10 张杂志图 | 从一亿个可能配对里，先粗筛出一个候选名单 |
| 2. 对应点匹配 | ASpanFormer | 数两张图之间能一一对上的细节点，少于 50 个就淘汰 | 类似指纹比对：能对上的细节点越多，越可能同源 |
| 3. 几何证据 | VGGT | 计算内点率、相机位姿等几何量 | 检查那些"对上的点"在几何上是否自洽 |
| 4. 判定 | 手工阈值 | 内点率 ≥ 0.65 且位姿分 ≤ 2.13 就判为匹配 | 两条人工划定的及格线 |

**实际的数量漏斗**（以第一批数据为例）：7,000 张原照 → 70,000 对候选 → 约 650 对通过第 2 步 → 由您人工过目 → 第 3、4 步判定。

**这条流水线的现有成绩**：在第二批数据上 F1 = 0.941（F1 是查准率和查全率的综合指标，1.0 为满分）。这个数字很好，问题只在于说不清楚。

**三个说不清**：

1. 阈值 50、0.65、2.13 都是试出来的，没有原则性依据。
2. 被第 2 步淘汰的 69,000 多对里有没有真配对？没人知道，也没人能全部人工检查。
3. 对 AI 会议而言，一条纯手工调参的流水线容易被归为"工程应用"而非"研究贡献"。

# 第三部分：新的形式化（阶段二的理论核心）

这部分是孔德乾的 formulation note 的内容，用非技术语言重述。

## 3.1 隐藏结构：两层分组

设想所有图像（原照和印刷品放在一起）构成一个集合。有两层看不见的分组：

- **曝光层**：一次快门产生一张底片；同一张底片的所有复制品（原照扫描件、杂志上的印刷版）属于同一个"曝光组"。**这是我们最终要找的关系。**
- **场景层**：同一次拍摄活动（同一地点、同一时段）产生的所有曝光，属于同一个"场景组"。

两层是嵌套的：同一底片必然同场景，反之不然。

这样定义有三个直接好处：

1. **传递性是免费的**：如果 A 和 B 是同一底片、B 和 C 是同一底片，那么 A 和 C 必然是同一底片——不需要任何人再看一眼。这在数学上是分组关系的固有性质，不是我们额外加的假设。这类"推导出来的关系"后面称为**闭包关系（closure）**。
2. **"同场景不同底片"有了正式位置**：它不再是"判错了的负例"，而是一个独立的类别。
3. **任务有了明确的估计对象**：我们要估计的是这两层分组结构，而不是"每一对图片打个分"。

## 3.2 三个层次的目标

德乾在 note 里定义了三个由易到难的估计目标，这也是论文实验部分的组织主线：

- **Q1**：候选配对的关系判定（有专家标注，可直接测量）。
- **Q2**：漏斗触及到的那部分图像的分组结构（部分可识别，随着关系网变密而变好）。
- **Q3**：全部图像的配对关系（端到端，需要估计漏斗漏掉了什么）。

## 3.3 最关键的一步棋：把"漏掉了什么"变成可测量的量

前面说过第二个"说不清"：被淘汰的 69,000 对里可能有真配对，但没人能全查。

新形式化的处理方式是**不辩护、改测量**。把步骤 1、2（检索 + 对应点门槛）整体宣布为**固定的、给定的候选生成装置**，我们不声称它最优。然后把最终查全率拆成两个因子相乘：

**端到端查全率 = 漏斗覆盖率 × 决策查全率**

- **决策查全率**：真配对进入候选名单后被正确接受的比例。有专家标注，可以严格测量。
- **漏斗覆盖率**：真配对能进入候选名单的比例。无法直接测量，但可以**抽样审计**：(a) 从被淘汰的配对中分层抽样请专家复核；(b) 用建模之前人工找到的已知配对当"考题"，看漏斗能否召回它们。

这样论文里就不存在"请相信 50 是对的门槛"这句话。淘汰造成的偏差（术语：**发现偏差**）从需要辩解的软肋，变成一个被定义、被测量、被如实报告的量。**这是整个形式化最有说服力的部分。**

## 3.4 决策层：机器不拍板

模型输出的是概率，不直接给"是/否"。最外层有一个校准过的决策规则，把每对候选分到三个筐：

- **接受**（几乎确定是真配对）
- **送人工复核**（拿不准，请专家看）
- **拒绝**（几乎确定无关）

并且带数学保证：**真配对被误扔进"拒绝"筐的比例不超过事先设定的上限 α**，同时让需要专家过目的量尽可能少。这个技术叫 conformal prediction（保形预测）。

对人文学界，这是最诚实的产品形态：机器不取代专家判断，只把专家的时间用在刀刃上，并保证漏检率可控。

**注意：这一层至今尚未实现。**这是当前的头号待办事项（见第八部分）。

# 第四部分：新的模型 NEG-Net

## 4.1 它是什么

一个小型图神经网络。"图"在这里指"点和线构成的关系网"：每张图片是一个点，每对候选配对是一条线。

它的工作方式：读取每条线上的证据（内点率、对应点数量、位姿差异等十几项现成指标），同时参考这个点的其他连线情况，输出两个概率——这对是同一底片吗？这对是同一场景吗？

规模约 150 万参数。在深度学习里这算**非常小**，是刻意的：我们只有几百对专家标注，模型太大就会死记硬背而不是学习规律。

## 4.2 与 Jordan 最初设想的关系

Jordan 最初的提议是训练一个简单的 MLP（多层感知机），输入是各阶段的特征，输出是二元判断。这个提议没有被否定——它成为了 NEG-Net 内部读取"每对证据"的那个部件（边编码器）。NEG-Net 在它之上增加了"参考邻居关系"的能力（消息传递）。

模型保留了一个开关，可以把消息传递**完全关掉**，此时它就退化成 Jordan 最初设想的版本。这个开关是论文归因表的基础：手工规则 → 简单模型 → 加消息传递 → 加一致性约束，每一步涨了多少，一目了然。审稿人最认这种逐项归因的证据。

## 4.3 三个配置（论文里反复出现）

| 代号 | 含义 |
|---|---|
| `mp0` | 关掉消息传递 = Jordan 最初设想的简单模型 |
| `mp3` | 完整版：消息传递 + 一致性损失 |
| `mp3_noloss` | 有消息传递，但去掉一致性损失 |

"一致性损失"指训练时的一条软性约束：如果 A=B、B=C 都很确定，模型却说 A≠C，就扣分。它把 3.1 节说的传递性变成训练信号。

## 4.4 输入的技术细节（了解即可）

- **每张图的数字指纹**：用现成 DINOv3 模型的原始输出（1024 个数字）。为什么用"加工前"的：我们自己训练的那个加工部件（fusion head）在消融实验中被证明可有可无，用原始版就不受它去留的影响。
- **每对配对的证据**：26 项统计量（对应点数、内点率、重投影误差、重叠比例、单应矩阵、位姿分量、整体相似度），加上"这项数据是否缺失"的标记，共 53 维。全部来自现有流水线的输出，**不需要任何新计算**。

# 第五部分：全部实验结果

这一部分是全文最重要的内容。所有数字来自 Jordan 编制的报告，每个数字都可追溯。

## 5.1 数据划分

| 数据 | 角色 |
|---|---|
| Shard 1（第一批） | 训练 |
| Shard 2（第二批） | 冻结验证（模型训练时完全不碰） |
| Shard 3–6 | 后备，Shard 3 保留作一次性盲测 |
| 晋察冀语料（沙飞档案 + 《晋察冀画报》） | 跨语料零样本测试，全程冻结后才碰 |

## 5.2 结果一：训练与跨批验证

在第一批上训练，在第二批（模型从未见过的图像）上冻结权重评测：

| 配置 | 第一批（自身验证折） | 第二批（跨批） |
|---|---|---|
| `mp0` | F1 0.953 | F1 0.920 |
| `mp3` | F1 0.958 | **F1 0.942** |
| `mp3_noloss` | F1 0.963 | F1 0.935 |

跨批只掉 1.6–3.3 个百分点，查全率保持在 0.974–0.977。这是好结果：说明模型学到的不是死记硬背。

## 5.3 结果二：与手工规则的正面对比（关键结果）

这是全项目最重要的一张表。在**完全相同的 479 条关系**上，把手工规则和 NEG-Net 放在一起比。

关系分两类：
- **449 条候选关系**：真实检索到并算过几何证据的配对，两种方法都能判。
- **30 条闭包关系**：由传递性推导出来的配对，**没有任何直接证据**。手工规则是逐对打分的规则，面对没有测量过的配对，它在结构上无法作答。

| 范围 | 方法 | F1 |
|---|---|---|
| 全部 479 条 | 手工规则 | 0.903 |
| 全部 479 条 | **NEG-Net (mp3)** | **0.942** |
| 仅 449 条候选 | **手工规则** | **0.959** |
| 仅 449 条候选 | NEG-Net (mp3) | 0.935 |

闭包关系上：手工规则 0/30，NEG-Net 三个配置全部 30/30。

**如何解读**：NEG-Net 在总体上领先，**完全是因为它能判定手工规则根本无法作答的那 30 条**。在双方都有证据的公平范围内，**手工规则更强**。

统计显著性检验（McNemar + Holm 校正）结论：在公平范围内手工规则的优势是稳健的；在全部范围内两者差异不显著。

## 5.4 结果三：跨语料零样本迁移（决定性实验）

把冻结的模型和冻结的手工阈值，原封不动搬到晋察冀语料上（沙飞档案 + 《晋察冀画报》，2,107 张照片 + 1,803 张印刷品）。这是最严格的测试：换了档案馆、换了年代、换了印刷工艺。

**金标准由您盲评产生**：951 对候选，其中 751 对由您人工裁定，**评审时看不到任何系统判决**。结果：771 对真配对，180 对非配对——其中 **75 对是"同场景不同底片"**（您自发标出的中间类）。

| 方法 | 查准率 | 查全率 | F1 |
|---|---|---|---|
| **手工规则** | 0.975 | 0.826 | **0.895** |
| `mp0` | 0.967 | 0.791 | 0.870 |
| `mp3` | 0.978 | 0.805 | 0.883 |
| `mp3_noloss` | 0.978 | 0.759 | 0.855 |

统计检验结论：
- 手工规则显著优于 `mp0` 和 `mp3_noloss`；
- **手工规则对完整版 `mp3` 的优势不显著**（p = 0.27）——在最难的测试上，差距缩小到统计上分不出高下。
- 闭包关系再次复现：NEG-Net 全部 104/104，手工规则 0/104。

**另一个重要现象**：所有方法的查全率都大幅下降（手工规则从 0.983 掉到 0.826）。Jordan 判断是晋察冀杂志的扫描质量明显差于《北支》。这说明**手工规则也没能无损迁移**，谁都没有赢下这场迁移测试。

## 5.5 结果四：两个属于我们的、统计上稳健的发现

| 假设 | 第二批（同语料） | 晋察冀（跨语料） |
|---|---|---|
| H1：消息传递有效 | 显著（p = 0.0089） | 显著（p = 0.030） |
| H2：一致性损失有效 | 不显著（p = 0.34），但方向为正 | **极显著（p < 0.0001）** |

**H2 是全文最有科学价值的单项发现**，理由是它符合"预注册"的标准：在第二批的报告里，我们**先**写下了理论预期（"正则化手段的真正考验是分布外泛化，不是同分布验证"），**然后**在更严格的跨语料测试上得到了压倒性验证。这不是事后从一堆结果里挑显著的，而是先预言后验证。

## 5.6 结果五：消融实验（流水线各阶段换零件）

- **第 1 步检索**：DINOv3-raw（R@1 98.3%）> SSCD（96.2%）> CLIP（84.6%）。三者并集能覆盖全部已知真配对。
- **第 2 步对应点**：LightGlue 与 ASpanFormer 统计上分不出高下（p = 0.60）。
- **第 3 步判定规则**：各种阈值组合的对比阶梯（B1/B2/B4/B5/B6 等）。

这些消融的作用是**论证候选生成装置的合理性**，与判定层的对比互不混淆（见 6.3 的"两平面"原则）。

# 第六部分：重大决策记录

以下每条都是过程中经过讨论确定的，记录在此以免遗忘理由。

## 6.1 更换检索骨干（7 月 19 日）

**决定**：Stage 1 从"自训练 fusion head 的 DINO 组合"改为"现成 DINOv3 原始权重"。

**理由**：(a) Jordan 的消融显示自训练部件可有可无；(b) 那个部件按设计需要在每个新语料上重训，会让"零样本迁移"的说法变复杂；(c) 换掉后全系统只剩 NEG-Net 一个学习部件，归因最干净。

**代价核算**：换骨干后 325 个已知正例丢 2 个、但新找到 3 个（净 +1，且新口径下覆盖 315 vs 313）。您审核了那 3 对新发现并确认无误。

## 6.2 数据泄漏的三次修补

这是项目中反复出现、Jordan 连续发现的问题。最终的完整规则：

1. **节点唯一归属**：每张图片只能属于一个批次。正例家族整体搬迁；只有负例连线的共享节点也按规则归批。
2. **诱导子图**：每批的关系网只保留两端都属于本批的连线，跨批连线（无论正负）从两边删除并如实报告数量。
3. **节点曝光登记**：模型训练时"看过"的每个节点被永久记录；任何后续评测集必须先删除这些节点。第二批评测中实际删除了 49 个节点、连带 278 条边（占 36.7%）。

**为什么必须这么严**：模型会记住见过的图像内容。训练时见过、评测时又出现，成绩就是虚高的。审稿人一旦发现训练集和测试集有同一张图，通常直接判定实验作废。

## 6.3 消融实验的"两平面"呈现原则

- **漏斗平面**（谁提供候选）：检索/对应点/几何模型的更换，用检索指标（Recall@K、覆盖率）呈现。
- **决策平面**（谁判定候选）：手工规则 vs NEG-Net 各配置，在**同一张冻结的关系网**上用 P/R/F1 呈现。

两个平面**永远不混用分母**。这解决了 Jordan "引入图模型后消融怎么呈现"的困惑，也避免了"每换一个流水线零件就要重训一次模型"的组合爆炸。

## 6.4 杂志图之间的配对：保留

您拍板：把杂志图也放进查询池是设计内的功能，因为我们也想知道**哪些杂志图彼此翻印自同一底片**。因此这类配对保留，并且它们带有完整几何证据。

## 6.5 截稿前不重训（重要纪律）

无论后续补多少标注，**截稿前绝不重训模型**。一旦重训，第二批、晋察冀、H1、H2 的全部结果作废重跑。全量审计后重训是 camera-ready 或后续论文的工作。

## 6.6 论文的诚实框架

主结果如实报告"手工规则在公平范围内更强"。这不是让步，而是符合我们自己形式化的逻辑：既然我们把查全率拆成两个因子并要求测量它们，就应当同样严格地对待自己的模型。摘要从未承诺"打败手工规则"——当时留的这个口子现在是有价值的。

# 第九部分：需要您知道的几个判断（我此前论证有误、已修正）

诚实起见，记录我在这个过程中修正过的两处判断：

**修正一：误报分层表的抽样论证**。我此前说"按相似度取前 150 对就能覆盖绝大部分误报"——这不严格，因为每个系统的"高分"口径不同。正确做法是直接导出"被任一系统接受过的负例"（定义上就是全部误报），不需要任何抽样论证。8.2 里的方案已按此修正。

**修正二：conformal 层的定位**。我此前说它是"为什么需要学习模型的最强答案"——这话过头了。审稿人可以反问"手工规则的连续分数也能加复核区间"，而且数据支持这个反问（手工规则的 PR-AUC 略高）。因此 conformal 层照做，但要**两边都套、同表报告**，不预设我们赢。"为什么要学习模型"的答案退回到三条真正站得住的：闭包可判性、H2 这个发现、无需逐语料手调阈值。

**一个新发现的漏洞**：所有 NEG-Net 结果目前都是单随机种子。这是 8.1 里 P1 项的由来。

# 第十部分：术语速查

| 术语 | 含义 |
|---|---|
| Shard（批） | 数据被分成的若干份，各自承担训练/验证/测试角色 |
| 曝光（exposure） | 一次快门 = 一张底片。同一底片的所有复制品属于同一曝光组 |
| 场景（scene） | 同一地点同一时段的拍摄活动，可能包含多次曝光 |
| 闭包关系（closure） | 由传递性推导出来的关系（A=B、B=C ⟹ A=C），没有直接证据 |
| 内点率（inlier ratio） | 对上的细节点中几何上自洽的比例，越高越可能同源 |
| 位姿（pose） | 算法反推的相机位置朝向；同一底片的两个印本应反推出几乎同一机位 |
| F1 | 查准率与查全率的调和平均，综合指标，1.0 满分 |
| PR-AUC | 排序质量指标，与具体阈值无关 |
| McNemar 检验 / Holm 校正 | 判断两个方法的差异是真实的还是偶然的统计方法；Holm 校正防止"多做几次检验总能撞上一个显著"的假象 |
| 消融实验（ablation） | 拆掉系统的某个零件再测，看它是否真有贡献 |
| 零样本（zero-shot） | 模型从未在该数据上训练或调参，直接测试 |
| 分布外（OOD） | 与训练数据不同来源、不同性质的数据 |
| conformal / 保形预测 | 一类能给出错误率数学上限的统计方法 |
| 发现偏差（discovery bias） | 标注只覆盖漏斗推荐过的候选，未被推荐的真配对天然不可见 |
| 数据泄漏（leakage） | 训练数据与测试数据有重叠，导致成绩虚高 |
| 预注册（pre-registration） | 先写下预期再做实验，避免事后挑选有利结果 |
