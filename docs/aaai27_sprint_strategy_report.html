The main-track call summarizes the difference: the track "emphasizes the fit between the techniques used and a problem of social importance, rather than simply rewarding technical novelty."

The single most dangerous policy change

AAAI-27's reviewer instructions now state that claims that code or data "will be released after acceptance or publication" must not be treated as evidence of reproducibility — reviewers are told to assume unreleased material will never exist. Your AAAI-26 submission promised NCR-Match "upon the camera-ready deadline" and it was never released; a returning reviewer can check that. 4The Five Fatal Issues (and the Majors)

Consensus of a three-lens review panel (AISI senior PC member · CV methods reviewer · DH applications reviewer) run against your actual documents, results, and repository.

Fatal F1 — Threshold leakage on Shard 1

The inlier-ratio cutoff (0.65), the pose-score threshold (2.13), and the pose-component weighting design itself were all tuned on Shard 1's hand labels — and Shard 1's P/R/F1 is then reported as a headline result. Shard-1 numbers are training-set numbers. This is the same sin ("thresholds chosen empirically on the eval data") that helped kill the AAAI-26 submission.
Fix: declare Shard 1 the dev set in the paper; label Shard 3 as an untouched test shard; report Shards 2–3 with everything frozen, plus sensitivity curves showing the operating point sits on a plateau, not a spike (§7).

Fatal F2 — The published numbers are irreproducible from the repo

The code audit's headline finding: the committed judge in vggt_judge.py:532-533 implements a different classifier (global-similarity ≥ 0.90 AND unweighted-L1 pose shift ≤ 0.10) than the one that produced the logged results (inlier-ratio 0.65 + weighted 9-dim pose heuristic at 2.13). The inlier-ratio filter does not exist anywhere in the repo; the pose weights are undocumented and live only in an off-repo "Experiment notebook." The public repository literally cannot reproduce the paper.
Fix (blocking, this week): commit the notebook heuristic as pose_scoring.py with the weights as named constants; acceptance test = reproducing F1 91.3 / 94.1 bit-for-bit from the stored manifests. If the notebook is lost, re-derive on Shard 1 only (it is the dev set anyway) and freeze by July 8.

Fatal F3 — Recall is measured on a selection-biased denominator

Labels exist only for pairs that already survived top-10 retrieval and the keypoint-count filter, so "R 96.3 / 98.4" is recall over pipeline survivors — the historian's real question ("what do I miss?") is unanswered, and retrieval-stage misses are invisible. The eval script compounds this: it counts needs_review/unlabeled pairs as false positives, so its precision is a different statistic from the hand-review precision, and nobody can currently say which produced 86.7/90.2.
Fix: (a) name the metric "verification P/R conditional on the candidate set" in the paper; (b) run the end-to-end recall audit — push the 2,636 verified NCR-Match pairs (the rebuttal's refined set) through every stage and report a per-stage survival table (§6.4); (c) reconcile the eval script with the hand-review protocol and report "Unsure" as a first-class outcome.

Fatal F4 — No expert validation, no measured impact

The ~1,279 labeled pairs were hand-reviewed by a single unnamed annotator with no codebook, no inter-annotator agreement, and "Unsure" silently excluded — while the methodology essay publicly claims the system collapses "months of manual comparison." For AISI, this is the difference between an application paper and a CV paper wearing a social-impact hat.
Fix: the two cheap studies in §6.6 — a multi-annotator relabeling with Cohen's/Fleiss' kappa under a written codebook, and a small within-subjects historian time-saved study. Start recruiting this week from the existing JOCCH annotation network.

Fatal F5 — Dataset and code release: promises are now worthless

See §2.2. NCR-Match remains unreleased a year after being promised; AAAI-27 reviewers are instructed to disregard promises.
Fix: deposit, don't promise. An annotation-only release avoids the image-rights problem entirely: pair IDs keyed to CODH's public North China Railway identifiers, match labels, trace annotations, the codebook, and a Gebru-style datasheet (provenance, labeling protocol, the archive's own Japanese-military selection bias, handling guidance for propaganda imagery). Email CODH / Harvard-Yenching / the Jinchaji Institute for written confirmation now.

5Two-Paper Strategy & the Agentic Question

5.1 Is the pipeline without the agentic network enough for AAAI?

A lesson the AAAI-26 reviews make unavoidable

Last year's rejection was an AISI rejection — and two of its four reviewers still wrote uncompromising CV-methods reviews. The track's tolerance for limited novelty did not save a paper whose Quality-of-Evaluation scores were Poor. Conclusion: whichever track you choose, the baseline/ablation program in §6–7 is non-negotiable; the AISI-specific additions (expert studies, release, findings, ethics) come on top of methods rigor, not instead of it. The encouraging flip side: significance scores were fine, and the two dimensions that sank the paper — evaluation quality and path-to-practice — are precisely the two this plan targets, with no new model required.

Main track: yes, cleanly — provided the ablation/baseline program in §6–7 lands. The exact-copy discriminator with a verified-novel pose signal, a rigorous multi-shard protocol, and the NCR-Match benchmark is a coherent, competitive main-track applications/methods paper. Adding an unevaluated VLM layer to it would weaken it: it dilutes the methods story, spends scarce pages, and hands reviewers a fresh soundness objection.

AISI without the agentic module: possible, but not with the current evidence. The panel's consensus: the pipeline alone, evidenced only by shard-level P/R/F1, is "a main-track paper wearing a DH hat" — the exact profile AISI reviewers are trained to reject. It clears the AISI bar only if the compensating package all lands:

  1. Expert validation with inter-rater agreement + the historian time-saved study (§6.6);
  2. The deposited benchmark (NCR-Match pairs + the new exact-vs-same-scene shard labels with trace annotations — arguably the first dataset separating identical exposures from same-scene negatives in cross-domain archival material, a headline AISI contribution by itself);
  3. 2–3 new humanities findings from the 42K corpus;
  4. The ethics section + misinformation-infrastructure framing.

5.2 What the agentic module is actually for

Through the DH lens, the VLM module is close to necessary for the AISI paper specifically — because the methodology essay itself observes that the routine, majority manipulation in these archives is textual ("what changes is the verbal envelope": recaptioning, name erasure, discursive re-placement), and the image-only pipeline is structurally blind to it. The module is the component that reads captions and page context; it converts binary match decisions into the provenance dossiers historians adjudicate from. But three constraints are non-negotiable:

  • It must not interpret. A VLM asserting "censorship" or "erasure" violates the project's published epistemics and creates a provenance-over-assertion liability in exactly the domain where the paper claims ethical care. Reports describe; historians interpret.
  • It must not re-derive what geometry already proved. Crop bounds, mirror flag, rotation angle come deterministically from the homography the pipeline already computed — reviewer-proof and free. The VLM contributes caption transcription/translation and neutral page-context description only.
  • It must be evaluated or it must not be claimed. The minimal rigorous evaluation is §8. If the expert grading cannot finish by July 28, demote the module to one qualitative case-study figure, explicitly not claimed as a contribution.

5.3 If you can only do one paper

Decision rule: the AISI paper is gated on human time (annotation campaigns + PI grading + partner sign-off on the data release), not GPU time. If by July 10 you cannot commit (a) a second annotator, (b) ~2.5 PI-days for grading/adjudication, and (c) written release confirmation from at least CODH, then submit Paper 1 to the main track only, run the expert studies properly over the autumn, and hold the AISI system paper for AAAI-28 — an AISI submission missing those elements would spend a rejection to learn what this report already says.

This converts the selection-biased "recall over survivors" into an application-level answer to "what does the historian miss?" — and per-stage loss attribution tells you where to spend future effort.

6.5 Pose-signal validity controls — making "pose-aware" mean something

  • Synthetic battery: take ~200 labeled exact-copy pairs; apply controlled crops, rotations, halftone simulation, mild perspective warps; show pose scores stay sub-threshold. This documents the degenerate picture-of-a-picture behavior as a measured feature rather than an assumption.
  • Per-component distributions: violin/ROC plots of each pose component (x/y-translation, rotation, fov, z) for exact-copy vs. same-scene labeled pairs — the empirical justification for the component weighting, currently asserted from anecdote.
  • Crop-stage QC audit (reviewer-requested): R3 asked for sensitivity to YOLOv11 cropping errors; the rebuttal answered ">98% detection, manually checked" without evidence. Formalize it: detection precision/recall on a 100-page audited sample, plus one sentence on failure handling when layout detection misses.

6.6 The human evidence (AISI-critical; start recruiting now)

  • Annotation reliability study: written 1-page codebook first (with the PI's ruling on contact-sheet adjacent frames and same-negative crops — the genuinely contested cases); then PI and the intern independently label the same pairs without seeing each other's answers, report Cohen's kappa, and adjudicate disagreements into gold labels, with Unsure as a first-class category. Scope: only the filter-survivor candidate pairs need human eyes — ≈1,279 pairs on Shards 1–2 (re-label) + ≈650 on Shard 3 (new) ≈ 1,950 pairs ≈ 2–3 working days per person. Shards 4–6 are out of scope this cycle — three labeled shards are enough for the paper's statistics.
  • Trace-type taxonomy: tag the ~577 confirmed positives by manipulation type from the essay's taxonomy — halftone-only, crop, rotation/mirror, retouch, recaption-with-near-identical-pixels — and report per-type pipeline pass rates. This operationalizes "the evidence of the small," and honestly exposes which class (recaption) the image-only pipeline cannot see — which is precisely the argument for the Paper-2 VLM module.
  • Historian time-saved study (within-subjects): 4–6 historians/grad students, 20–30 query photos each; condition A = pipeline-verified candidates with alignment visualizations, condition B = plain top-10 retrieval list (or manual browsing); measure time-to-first-verified-republication and verified matches per 30-minute session. Small N is fine — the track rewards real-world rigor, not scale. Fallback if recruiting fails: present the new case-study discoveries as the field evidence and state the user study as registered future work with named partners.

6.7 Cross-domain transfer: Hokushi → Jinchaji (the AISI showpiece)

Freeze everything (thresholds, weights, breakpoint) and run the pipeline on the JOCCH Jinchaji data (2,146 photos + 4,390 crops; 230 annotated pairs; ≈6 A100-h + 1 labeling day). Report Recall@10/15 against the JOCCH paper's 77.8% top-15, and verification P/R/F1 on hand-labeled survivors. What it proves: the identical pipeline and identical thresholds transfer from Japanese occupation propaganda (1939–45) to Communist base-area propaganda (1942–46) — two opposing propaganda systems, different printing technology, different degradation. It is the essay's generalization claim ("Soviet pictorials, colonial photography, Cold War print") made concrete, and almost no copy-detection paper has a cross-dataset result in this domain. Fallback: retrieval-only transfer (zero new labels).

6.8 Statistical reporting (applies to every table)

  • Bootstrap 95% CIs (10,000 resamples over pairs) on every P/R/F1/PR-AUC cell — cheap, no GPU.
  • McNemar exact test between the full pipeline and each variant on identical pairs (paired decisions are exactly what McNemar is for), Holm-corrected across the table.
  • 3 seeds for the final SSL model (mean±std); 1 seed for ablation variants, stated; 5 cv2-seed repeats on Shard 1 to demonstrate geometric-stage variance ≈ 0 after seeding.
  • Disclose the transductive SSL setup (the fusion head trains on the evaluation corpus) and run the one control: retrain excluding Shard-3 images, show Shard-3 F1 unchanged within CI.

7The Rigorous Ablation Protocol, Condensed

The answer to "怎么做比较严谨" in five rules:

  1. Declare the split roles in the paper. Shard 1 = development (all thresholds and the weighting design were chosen here — say so). Shard 2 = held-out validation (looked at once; never tune on it again). Shard 3 = test, labeled this sprint under the codebook, evaluated exactly once, in the final week, with everything frozen. Do not promise 6-shard statistics — only three shards will ever be labeled; report per-shard columns with CIs instead.
  2. One factor at a time, from cached artifacts. The pipeline is staged and manifest-cached by design — exploit it. Every threshold-level ablation (keypoint breakpoint, inlier ratio, pose score, component weights) is CPU post-processing over aspan_all_manifest.jsonl and vggt_judged_manifest.jsonl; only backbone swaps and matcher swaps need GPU reruns.
  3. Every claimed mechanism gets an isolating row. "Depth-aware" died in 2025 because it was never isolated. "Pose-aware" survives only with: pose-off (B2), pose-from-homography (B8), pose-components split (B5/B6), learned-combination (B11), and the validity controls (§6.5).
  4. Paired statistics, not point estimates. Bootstrap CIs on cells; McNemar between variants; sensitivity curves around every hand-set threshold. A reviewer should be able to see that 0.65 and 2.13 sit on plateaus.
  5. The honest denominators. Verification metrics are conditional on the candidate set — name them so. End-to-end recall comes only from the §6.4 survival audit. "Unsure" is reported, never silently dropped.

8The VLM Module, Done Credibly (Paper 2 Only)

8.1 Output contract

Strict JSON schema per pair — never free text:

  • verdict: exact-copy / same-scene-different-shot / uncertain — taken from the CV pipeline, not re-decided by the VLM;
  • geometric_edits: crop bounds, mirror flag, rotation angle, scale — computed deterministically from the homography, passed in as evidence;
  • editorial_traces: retouch / overlay / recaption — each {present, evidence-pointer};
  • caption: transcription + translation of the printed caption (kyūjitai-aware);
  • page_context: 2–3 neutral sentences on layout and the "discursive neighborhood";
  • provenance_fields: publication, issue, date, page as read from the page image;
  • Zero intent language. No "censored", no "erased", no "propaganda aims". Interpretation is the historian's.

Temperature 0; pinned model version string ("Gemini 3.1 Pro, preview, accessed July 2026"); 3 repeat calls on a 20-pair subsample to report output stability; one open-weights VLM (Qwen3-VL) run on the same schema for reproducibility.

8.2 Evaluation (the part that makes it publishable)

  • Sample: N = 100 stratified pairs — 40 true exact-copies (including heavy crops/halftone), 30 same-scene negatives, 15 pipeline errors, 15 Unsure.
  • The one ablation that matters: condition (i) VLM sees source photo + magazine page only; condition (ii) VLM additionally receives the CV evidence (inlier ratio, pose deltas, aligned overlay). Hypothesis: grounding reduces verdict errors and hallucinated traces. 200 reports total.
  • Expert grading protocol: PI grades blind to condition, randomized order — verdict correctness; each claimed trace ∈ {correct, incorrect, unsupported}; hallucination rate (unsupported factual claims per report); caption-transcription character accuracy on a 50-pair subset; and the interpretive-overreach rate — assertions of intent per report, target ≈ 0. The intern double-grades a 25-pair overlap for Cohen's kappa. Budget: ~4 min/report × 200 ≈ 13 h ≈ 2.5 PI-days.
  • Comparison: VLM-alone vs. CV-pipeline-alone vs. CV+VLM verdict accuracy on the same pairs (McNemar) — plus the qualitative capability the pipeline lacks: recaption/context traces.
  • Fallback: if PI time collapses, N=60 single-condition + a 20-pair grounding ablation; below that, demote to a qualitative figure and claim nothing.

Why the interpretive-overreach metric is worth having

It operationalizes the project's own published principle — "the pipeline does not interpret; it retrieves" — as a measurable property of an AI system. That is a quotable methodological contribution in itself, it is the direct answer to the AISI ethics expectation, and it differentiates the module from the FakeShield/ForenAgent wave of explainable-forensics work, which has no equivalent restraint metric.
